# 🎮 StandQuest Demo - 開發紀錄與專案說明

這是一個基於 Socket.io 的多人即時互動問答遊戲原型，設計概念類似「糖豆人」，玩家可以在地圖上自由移動並在指定時間內跑向正確答案區域。

## 🚀 如何啟動遊戲

1. **安裝依賴**：
   ```bash
   npm install
   ```
2. **啟動伺服器**：
   ```bash
   npm start
   ```
3. **開啟遊戲**：
   在瀏覽器中開啟 `http://localhost:3000`。

## 🎮 遊戲機制

- **目標**：根據螢幕上方顯示的問題，移動到你的角色跑向代表正確答案的區域（Zone A、B 或 C）。
- **操作方式**：
    - **PC**：使用 `WASD` 或 `方向鍵` 控制移動。
    - **手機/平板**：使用螢幕左下角的**虛擬搖桿**移動。
- **遊戲流程**：
    1. **LOBBY**：進入遊戲後在等候區閒逛，等待主持人（GM）開始。
    2. **QUESTION**：GM 按下「下一題」後，螢幕顯示題目與倒數計時。你需要在時間內跑到正確區域。
    3. **FREEZING**：時間結束，系統判定分數。答對者加分，答錯者無分。
    4. **結果揭曉**：短暫顯示結果後，自動進入下一題或回到 LOBBY。

## 📝 題目管理

### 方式一：GM 控制台（推薦）

開啟 GM 控制台 `http://localhost:3000/gm.html`，點擊「📝 題目編輯」即可：

- **新增題目**：選擇 2-3 個選項，設定計時
- **編輯題目**：修改題目內容、選項、正確答案
- **刪除題目**：至少保留一題
- **排序題目**：拖放調整題目順序
- **儲存全部**：自動偵測變更並同步

### 方式二：直接編輯 questions.json

題目存放在 `data/questions.json` 檔案中。你可以使用任何文字編輯器（如 VS Code）開啟並修改。

**題目格式範例**：
```json
{
  "id": "unique_id",
  "questionText": "這裡輸入問題內容",
  "options": ["選項 A", "選項 B", "選項 C"],
  "correctIndex": 0, 
  "timeLimit": 15
}
```

**欄位說明**：
- `questionText`: 顯示在螢幕上的問題。
- `options`: 選項陣列（對應下方的 Zone A, B, C）。
- `correctIndex`: 正確答案的索引值（0 代表 Zone A, 1 代表 Zone B, 2 代表 Zone C）。
- `timeLimit`: 倒數秒數。

### 動態答題區域

答題區域數量會根據題目選項數量自動調整：

| 選項數量 | 區域數量 | 區域佈局 |
|---------|---------|---------|
| 2 個選項 | 2 個區域（A/B） | 各佔 50% 寬度 |
| 3 個選項 | 3 個區域（A/B/C） | 各佔 33% 寬度 |

GM 新增題目時，選項數量限制為 2-3 個。

## 📋 GM 控制台功能

GM 控制台提供完整的遊戲管理功能：

### 遊戲控制
- **開始遊戲**：啟動遊戲流程
- **下一題**：手動觸發下一題
- **暫停/繼續**：暫停或恢復遊戲
- **強制結束**：立即結束當前題目

### 題目管理
- **📝 題目編輯**：完整的題目 CRUD 操作
  - 新增題目（2-3 個選項）
  - 編輯題目（內容、選項、計時）
  - 刪除題目
  - 排序題目
  - 儲存全部 / 恢復原始

### 即時狀態
- 遊戲狀態徽章（LOBBY / QUESTION / FREEZING / RESULT_DISPLAY / PAUSED）
- 目前題號與總題數
- 在線玩家數量
- 倒數計時器
- 題目預覽
- 玩家列表（暱稱、分數、位置）

### 🛠️ 技術細節
- **Express JSON middleware**：已啟用 `express.json()` 以支援 POST API。
- **Render 相容性**：`fs.writeFileSync` 加入 try-catch 容錯處理，避免 Render ephemeral filesystem 導致 500 錯誤。
- **動態 zones**：`getZonesForQuestion()` 函數根據題目選項數量動態生成答題區域。
- **邊界檢查**：`handleTimerEnd()` 加入 `targetZone` 邊界檢查，防止 correctIndex 超出範圍時 crash。

## 🕹️ 遊戲玩法說明
1. **加入遊戲**：玩家輸入暱稱並點擊「開始遊戲」。
2. **等待指令**：在 LOBBY 階段，玩家可在地圖上自由走動。
3. **搶答挑戰**：當 GM 啟動題目時，螢幕上方會出現問題與倒數計時。玩家必須迅速跑向代表正確答案的彩色區域（例如：若答案是 B，則需站在 Zone B）。
4. **勝負判定**：時間結束後，留在正確區域的玩家將獲得加分，並在下一題開始前看到統計結果。

## 📂 專案結構說明
- `server.js`: 後端核心邏輯（遊戲狀態、Socket 事件處理、API 端點）。
- `public/game.js`: 前端遊戲循環（Update/Draw）、輸入控制與 Socket 接收。
- `public/index.html`: 遊戲主入口與 UI 介面。
- `data/questions.json`: 題目資料庫，可隨時擴充題庫。
